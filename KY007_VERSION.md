## 🎯 问题分析

### 问题描述
**用户反馈**：
- 输入：`我就接收到几道`（6字）+ `不善的视线`（5字）= 11字
- 期望：合并成一行（11字 ≤ 12字）
- 实际：保持两行，未合并

### 问题根源

**问题 1：缺少换行符拆分**
- `PUNCTUATION` 数组中没有包含 `\n`（换行符）
- `splitByPunctuation` 函数按标点符号拆分时，不会拆分换行符
- 结果：多行文本被当作一行处理

**问题 2：缺少合并短行逻辑**
- 当前流程中，删除标点后直接进入拆分流程
- 没有检查相邻短行是否可以合并
- 结果：即使相邻行的总字数 ≤ maxChars，也不会合并

### 用户期望
- 相邻的短行应该合并，如果总字数 ≤ maxChars
- 目标是让每行尽可能接近 maxChars，而不是只有几字

## 🔧 解决方案

### 核心思路
1. 在 `PUNCTUATION` 数组中添加 `\n`，支持按换行符拆分
2. 在删除标点后、拆分超长行前，新增"合并短行"步骤

### 技术实现

**1. 添加换行符到标点符号数组**：
```typescript
const PUNCTUATION = ['，', '。', '！', '？', '；', '：', ',', '.', '!', '?', ';', ':', '、', '（', '）', '(', ')', '【', '】', '「', '」', '『', '』', '《', '》', '…', '……', '\n'];
```

**2. 新增合并短行函数**：
```typescript
function mergeShortLines(lines: string[], maxChars: number): string[] {
  const result: string[] = [...lines];
  let merged = true;

  // 循环合并，直到无法再合并为止
  while (merged) {
    merged = false;

    for (let i = 0; i < result.length - 1; i++) {
      const currentCharCount = countChineseChars(result[i]);
      const nextCharCount = countChineseChars(result[i + 1]);
      const totalCharCount = currentCharCount + nextCharCount;

      // 如果当前行和下一行的总字数不超过限制，合并它们
      if (totalCharCount <= maxChars) {
        result[i] = result[i] + result[i + 1];
        result.splice(i + 1, 1);
        merged = true;
        break; // 重新开始遍历
      }
    }
  }

  return result;
}
```

**3. 在流程中插入合并步骤**：

## 📝 技术细节

### 合并算法
**循环合并策略**：
1. 遍历所有相邻行对
2. 检查两行字数之和是否 ≤ maxChars
3. 如果是，合并两行并重新开始遍历
4. 如果否，继续检查下一对
5. 重复直到无法再合并为止

**为什么需要重新开始遍历**：
- 合并后，新行可能与下一行再次满足合并条件
- 需要重新遍历以确保最优合并结果

### 性能影响
**新增步骤**：步骤3.5 - 合并相邻的短行
- 时间复杂度：O(n²)，其中 n 是行数
- 实际影响：对于正常文本（< 100行），几乎无感知
- 优化：已添加日志，便于监控性能
