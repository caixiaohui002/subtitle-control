## 🎯 问题分析

### 问题描述
**用户反馈**：
- 输入："以内部高息理财的名义"（10字）
- 期望：不拆分（10字 ≤ 12字）
- 实际：被拆分成两行 "以内部高息理财\n的名义"

### 问题根源
**LLM 拆分逻辑缺陷**：
- 在 `splitWithLLM` 函数中，LLM 被要求"按照语义边界将文本拆分成多行"
- **关键问题**：LLM 的系统提示词中没有明确限制"只有在字数超过限制时才拆分"
- 导致 LLM 即使在字数 ≤ maxChars 时也会按语义边界拆分

**示例分析**：
- 输入："以内部高息理财的名义"（10字）
- LLM 认为：两个语义单元（"以内部高息理财" + "的名义"）
- LLM 动作：按照语义边界拆分
- 结果：拆分成两行，虽然字数未超限

## 🔧 解决方案

### 核心思路
**在调用 LLM 之前，先检查字数是否超过限制**：
- 如果字数 ≤ maxChars：直接返回原文，不调用 LLM
- 如果字数 > maxChars：调用 LLM 进行语义拆分

### 技术实现

**修改 `splitWithLLM` 函数**：
