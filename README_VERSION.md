# KY001 版本管理指南

## 📋 概述

KY001 是一个经过充分测试的稳定可用版本，包含完整的智能文本处理功能。本文档说明如何使用版本管理和回滚功能。

## 🎯 KY001 版本功能清单

### 核心功能
- ✅ 删除报幕信息（支持多种格式）
- ✅ 标点符号断行
- ✅ LLM 语义拆分
- ✅ 自定义每行字数
- ✅ 流式响应（实时进度）
- ✅ 取消处理功能
- ✅ 拖拽上传
- ✅ 复制/下载功能
- ✅ 实时调试日志

### 支持的报幕格式
- `剧名-第1集`（传统格式）
- `剧名-EP1`（英文字母格式）
- `剧名-Season1`（季数格式）
- `剧名-1`（纯数字格式）
- `剧名 - 第1集`（带空格，自动容错）

## 📦 版本文件说明

### 1. KY001_VERSION.md
版本详细记录文件，包含：
- 版本信息（名称、类型、创建时间）
- 核心功能列表
- 技术栈
- 关键文件列表
- 测试用例
- 启动方式
- 回滚方法

### 2. backup-ky001.sh
备份/恢复脚本，用于：
- 快速备份当前版本
- 快速恢复到 KY001 版本

### 3. /workspace/projects/backup/KY001/
备份目录，包含 KY001 版本的关键文件：
- `page.tsx`（前端主页面）
- `route.ts`（后端 API）
- `package.json`（依赖配置）
- `.coze`（项目配置）
- `backup_time.txt`（备份时间戳）

## 🚀 使用方法

### 查看版本信息
```bash
cat /workspace/projects/KY001_VERSION.md
```

### 备份当前版本到 KY001
```bash
cd /workspace/projects
./backup-ky001.sh backup
```

### 恢复到 KY001 版本
```bash
cd /workspace/projects
./backup-ky001.sh restore
```

重启服务：
```bash
coze dev
```

### 查看备份文件列表
```bash
ls -lh /workspace/projects/backup/KY001/
```

## 🔙 回滚场景

### 场景 1：新版本有严重 Bug
1. 使用脚本快速恢复：
   ```bash
   ./backup-ky001.sh restore
   ```
2. 重启服务验证功能
3. 修复 Bug 后再重新测试

### 场景 2：误删或修改了关键文件
1. 从备份目录手动恢复单个文件：
   ```bash
   cp /workspace/projects/backup/KY001/page.tsx /workspace/projects/src/app/page.tsx
   ```
2. 检查文件是否正确恢复
3. 重启服务

### 场景 3：功能回归测试
对比 KY001 功能点与当前版本，确保核心功能：
- 报幕删除功能正常
- 标点断行功能正常
- LLM 语义拆分正常
- 取消功能正常

## 📝 版本更新流程

### 创建新版本
1. 在当前版本基础上开发新功能
2. 充分测试新功能
3. 确认功能正常后，创建新的版本文件
4. 更新备份脚本（如需要）
5. 记录版本变更

### 建议命名规则
- KY001 - 初始稳定版
- KY002 - 第二个稳定版
- ...

## ⚠️ 注意事项

1. **重要修改前务必备份**
   - 修改关键文件前先备份
   - 或使用 Git 进行版本控制

2. **测试后再覆盖**
   - 修改后先测试
   - 确认无误再考虑更新 KY001

3. **记录变更**
   - 重大修改更新版本记录
   - 记录修改原因和影响

4. **定期备份**
   - 定期执行备份命令
   - 保留多个历史版本

## 📞 故障排除

### 恢复后服务无法启动
1. 检查端口占用：`ss -tuln | grep 5000`
2. 检查依赖安装：`pnpm install`
3. 检查配置文件：`cat .coze`
4. 查看日志：`tail -n 50 /app/work/logs/bypass/app.log`

### 恢复后功能异常
1. 检查文件是否完整：`ls -lh backup/KY001/`
2. 检查文件权限：`ls -l src/app/page.tsx`
3. 重新执行恢复：`./backup-ky001.sh restore`

### 备份脚本无法执行
1. 检查执行权限：`ls -l backup-ky001.sh`
2. 添加执行权限：`chmod +x backup-ky001.sh`
3. 检查脚本格式：`cat backup-ky001.sh`
