## 🎯 问题分析

### 用户反馈
**用户明确需求**：
- 不要"智能合并"
- 要的是"不超过12个字的行数不调用语义直接断行"

### 之前的错误（KY007）
我在 KY007 中添加了"合并短行"功能，这**不是用户需要的**！

**错误的假设**：
- 我以为用户想要合并相邻的短行，让每行字数更接近 maxChars
- 所以添加了 `mergeShortLines` 函数

**实际用户需求**：
- 保持原有的行结构，不合并
- 每行独立检查字数
- 如果 ≤ maxChars：保持原样，不调用 LLM
- 如果 > maxChars：调用 LLM 进行语义拆分

## 🔧 修正方案

### 核心逻辑（正确的）
1. 按标点符号和换行符拆分
2. 删除所有标点符号（包括换行符）
3. 检查每一行：
   - 如果字数 ≤ maxChars：**保持原样，不调用 LLM**
   - 如果字数 > maxChars：调用 LLM 进行语义拆分
4. **不合并相邻的短行**

### 技术实现

**1. 删除 mergeShortLines 函数**：
```typescript
// ❌ 删除这个函数
function mergeShortLines(lines: string[], maxChars: number): string[] {
  // ...
}
```

**2. 删除步骤3.5（合并短行）**：
```typescript
// ❌ 删除这个步骤
// ==================== 步骤3.5: 合并相邻的短行 ====================
const mergedLines = mergeShortLines(linesWithoutPunctuation, maxCharsValue);

// ✅ 直接使用 linesWithoutPunctuation
const finalLines: string[] = [...linesWithoutPunctuation];
```

**3. 保留 \n 在 PUNCTUATION 中**：
```typescript
const PUNCTUATION = [..., '\n'];
```
- 目的：让换行符作为分隔符
- 效果：按换行符拆分后，删除 \n，变成真正的换行
